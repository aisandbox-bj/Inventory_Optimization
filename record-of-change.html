Record of Change

Every release is tagged on the GitHub repo and listed here. Each entry names what changed, what schema/file boundaries it touched, and the exact git command to roll back to any prior version. Releases marked dev are in-flight; they become released when the tag lands on main.
How to roll back

Reset to any tagged version

From a clone of the repo (e.g. /tmp/push-Inventory_Optimization), check out any tag to see the working tree at that version, then either copy the files back into your local App/v1/ directory or reset main to that tag. The user's working folder is not a git repo — rollback always happens through a clone, never via git init in the working dir. 

# Inspect a prior version (read-only checkout)
git clone https://github.com/aisandbox-bj/Inventory_Optimization.git /tmp/peek
git -C /tmp/peek checkout v1.0.0
# → now /tmp/peek holds the working tree at v1.0.0; copy what you need back

# Or hard-reset main to a tag (irreversible — only when intended)
git -C /tmp/peek checkout main
git -C /tmp/peek reset --hard v1.0.0
git -C /tmp/peek push --force-with-lease origin main

Local App/v1/ reflects whatever Claude has just written — it tracks the dev tip until the next release tag. Save a snapshot before destructive edits if uncertain.

v1.1.0 2026-05-11 dev

Intake — Assessment Type selector

  • New Step 0 on the Intake page: pick one of UNIT / FLOC based, User specified List, or Parameter search.
  • Upload zones not required by the selected type are greyed out and reject drops, eliminating ambiguity about which data is needed for which workflow.
  • Selection persists in canonical JSON as metadata.assessmentType.

Intake — Inventory Master extended fields

  • Parser now resolves Material Group, Manufacturer, Value on hand (Stock value), and keeps existing Qty on hand, MRP indicator, Inventory Type — through column aliases, so SAP export drift remains tolerable.
  • Fields are optional: a missing column simply removes that filter dimension from Parameter Search.

Parameter Search — PBI-style filter panel

  • When assessment type = Parameter search, an inline filter panel appears after upload and before "Save & Continue".
  • Drag dimensions from a pill bar onto the canvas to add filters. Multiple filters AND together.
  • Each filter supports Simple (multi-select values, or numeric range presets) and Advanced (custom min/max, regex, or comma-separated value lists).
  • Live preview: matching material count + total net consumption update as filters change.
  • Dimensions: MB51 movement (net qty), Material Group, Manufacturer, Value on hand, Qty on hand, Inventory Type, MRP indicator.

Version management

  • This Record of Change page added and linked from every page's nav.
  • Bumped APP_VERSION to 1.1.0-dev in shared/canonical-schema.js.
  • Prior fully-working baseline tagged as v1.0.0 on GitHub for rollback.

v1.0.0 2026-05-11 released

First fully-working baseline

  • Phase 1 — Foundations: brand tokens, canonical schema v1.0.0, storage helpers, SAP parser with column-alias auto-mapping.
  • Phase 2 — Intake engine: 6-step workflow (upload → schema → DQ → scope → params → export). Hybrid DQ gate (green/amber/red) ports 01_data_quality.py.
  • Phase 3 — Settings tab: persistent parameter defaults, LLM provider keys + dynamic model fetch, column-alias overrides.
  • Phase 4 — Analysis engine: deterministic pipeline (period rates, HCE, lumpy/smooth, 8-rule traffic-light), inline SVG cumulative chart with P1/P2 trend chords, provider-agnostic LLM review surface, ExcelJS workbook export with embedded charts.

Fixes folded into v1.0.0

  • Excel export crash (worksheet.column → worksheet.getColumn) on Summary + Run metadata sheets.
  • Combined-workbook export added: single file with Master Index, per-bucket sheets, cross-bucket Summary, Run metadata.
  • Chart trend lines re-anchored to actual cumulative values at both endpoints (chord across the period) with visible markers and bolder rate labels.
